Add the plugin to an existing project

Open an existing project

  1. To launch Unreal Engine, open the Epic Games launcher and click on Library.

  2. Choose the correct version (4.27.x) and click Launch. Select the project from the Recent Projects area. If your project is not displayed under Recent Projects, click More to display all available projects. Click Open Project.

    Tip

    If you want to bypass the Unreal Project Browser, you can click More to expand the Recent Projects area, and check the box for Always load last project on startup. The last project you worked on will now open when you launch UE4.

Install the plugin

The ArcGIS Maps SDK for Unreal Engine plugin is delivered as a zip archive and available on the Esri Early Adopter site.

To install the plugin:

  1. Exit UE4. Then create a Plugins folder in the main project directory.

    Note

    The folder name must be plugins or Plugins.

  2. Extract the ArcGISMapsSDK folder from the .zip file and copy it to the new Plugins folder.

    Note

    If you encounter The path is too long error, extract the .zip first and copy it into the Plugins folder, instead of copying the .zip file to the Plugins folder

  3. Start UE4.

Manage existing content

To allow your project’s existing Actors to track within an ArcGISMap with geographic coordinates, you can add an ArcGIS Location component.

You can find the script here:

ArcGISMapsSDK C++ Classes\ArcGISMapsSDK\Public\ArcGISMapsSDK\Components

To use the ArcGISLocation component:

  1. Click Add Component in the Details Panel and search for ArcGIS Location to add the component to your Actors. For more information on components see Components.

  2. Select your Actors and change the Mobility in the Details Panel to Movable.

  3. Click on the ArcGIS Location in the component and set the initial position and rotation. The ArcGISLocation component uses the pivot of the Actor as the setting point.

    • Latitude: Sets the north–south position of the Actor on the Earth's surface.
    • Longitude: Sets the east–west position of the Actor on the Earth's surface.
    • Altitude: Sets the vertical distance of the Actor above the Earth.
    • Heading: Sets the heading or up axis of the Actor. This is also known as yaw or azimuth. It represents the number of degrees by which the game object will be rotated, measured counterclockwise from the north. To rotate clockwise, use a negative value.
    • Pitch: Sets the pitch or right axis of the Actor. Pitch is the equivalent of moving a plane's nose up or down (rotating along the axis that passes through the plane's wings). Positive values, looking forward, are above the horizon (pointing upward) and negative values are below the horizon (pointing downward).
    • Roll: Sets the roll or forward axis of the Actor. Roll is the equivalent of tilting a plane's wings up or down (rotating along the axis that passes through the center of the plane from front to back). A zero value is perfectly horizontal. Positive values will tilt the left side upward (or right side downward). Negative values do the opposite.
    Tip
    • You do not need to add the ArcGIS Location component to every Actor in the level. Instead, you can create an empty Actor, attach the Location component to it, and additional Actors can be parented to the Actor.
      • This is useful for when you need to bring multiple meshes into your ArcGIS scene for a player level.
    • If you cannot find your object after setting the location, try to adjust the scale of the object. Make sure you adjust the scale of the object by selecting the root object and not the ArcGIS Location component.
    • Some 3D models may have different axis orientations when they are imported from different 3D modeling software. If your 3D model is rotated in the wrong direction, you can adjust the Rotation properties in the ArcGIS Location component's Details panel.
